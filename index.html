05 · Test ideas

Suggestions to get you started

The lists below are just starting points — not a checklist to tick off. Be creative: try odd state combinations, race conditions, weird inputs, and flows we didn't think of. The unexpected paths are where the interesting bugs live.

Group 1 · Payroll

Payroll flows

Payroll Onboarding

  • App should be blocked if the user hasn't completed onboarding
  • Should not block if the facility doesn't have payroll enabled, or the user is already onboarded
  • Redirect to Schedule after onboarding completes
  • Multiple facilities

Payroll Settings

  • View information
  • Edit information

Payroll Documents

  • View documents
  • Download documents

Earnings

  • Payroll enabled: show worked hours and paystub values per pay period
  • Timekeeping on, payroll off: show timecards and hours based on timecards
  • Both off: show shifts and hours based on shifts
  • Payroll enabled: each paystub must be downloadable individually
  • Screens show correct values for net pay, time-offs, and worked hours
Group 2 · Punches

Punch flow

Timekeeping disabled

  • Punches should not be possible — no clock-in / clock-out / break buttons visible

Timekeeping enabled — Kiosk only

  • No punch options shown in the app (clock in / break / clock out)
  • Always show text directing the worker to the kiosk
  • After clock in, show the punch timeline screen

Timekeeping enabled — in-app punches

  • Punch action buttons appear (clock in, breaks, clock out)
  • Presence verification off: punch without verifying location
  • Presence verification on: enter the location verification flow on every punch
  • Signature off: don't require a signature to complete the timecard
  • Signature on: require a signature after clock out to complete the timecard
  • Missing clock in: still allow clock out once inside the clock-out window
  • Shift status displays correctly and matches the clock-in/out window and timecard state (in progress, break in progress, shift starts in…)

Standalone clock-in

  • Enabled: show the option to clock in without a scheduled shift
  • Disabled: no unscheduled clock-in button appears
  • Enabled + multiple units: bottom sheet appears to select the unit after clock in

Breaks

  • Auto break: user can't start a break manually
  • Auto break: timecard timeline doesn't show break times
  • No auto break: start break / skip break options appear
  • No auto break: break times show on the timeline
Group 3 · Time-offs

More screen & support

Time-off screen

  • View upcoming time-offs
  • View past time-offs
  • View available time-off hours / balances
  • Facility dropdown shows up for users with multiple facilities
  • Associated plan is visible on the balances screen

Time-off details

  • Screen opens when tapping a time-off item in the schedule
  • Redirect to the all time-offs screen works
Reminder: these are suggestions, not requirements. Use them as a warm-up, then go off-script — the goal is to surface what we haven't anticipated.
